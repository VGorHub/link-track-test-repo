# link-track-test-repo

## Демонстрация AI-суммаризации

Этот раздел добавлен для демонстрации функции автоматической суммаризации обновлений в системе LinkTracker. Когда пользователь отслеживает репозиторий GitHub, сервис scrapper периодически проверяет изменения и фиксирует новые события, связанные с репозиторием. После обнаружения обновления информация передается в дальнейший конвейер обработки уведомлений.

Сервис ai-agent анализирует полученный текст обновления, отбрасывает второстепенные детали, определяет важность события и формирует более короткое описание. Это позволяет не отправлять пользователю слишком длинные технические сообщения, особенно если изменение связано с большим описанием pull request, issue, commit или обновлением файла README.

В данном примере файл README специально дополнен подробным описанием работы системы. Ожидаемый результат состоит в том, что LinkTracker обнаружит изменение репозитория, передаст длинный текст в ai-agent, а пользователь получит краткое и понятное уведомление через Telegram или другой настроенный канал доставки.
